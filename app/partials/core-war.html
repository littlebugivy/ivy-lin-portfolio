Role : Team leader, game designer, front-end developer
Applied : User case study, Quick Prototyping, Keynote, Android Studio, Java, XML

Core War is a programming PC game firstly established in 1984. In the game, two or more battle programs (or we call them "warriors") compete for control of one virtual computer. These warriors are written in an abstract assembly language called Redcode, which is designed for Core War particularly. Therefore fundamental knowledge of assembly language programming is required to play this game.

At the beginning of a game, each battle program is loaded into memory at a random location, after which each program executes one instruction in turn. The objective of the game is to cause the processes of opposing programs to terminate (which happens if they execute an invalid instruction), leaving the victorious program in sole possession of the machine.

Our mission is to bring the game into the mobile era, allowing broader access for knowing, studying and playing this game.

Needs Identification

Step 1:Persona

To narrow down the software functionality, we considered various user cases and sketched out personas which represent two typical categories of target users: Core War novice and veteran.

Step 2: Competitive Analysis

As there are no mobile adaptations of the game on either Android or iOS platforms so far, we need to seek for our own way for game development on the mobile end. The classic version of Core War contains considerable settings as well as functionalities, which gives rise to great complexity and difficulty of development. Considering not all the funcationalities are necessary for players normally, also due to the time limitation, we eventually decided to simplify the game while kept essential game logics. To get a better understanding of the game and make a decision on which settings should be remained or removed, we studied the previous software by trying them out and also investigated several experienced Core War players for advice.



Step 3: Specification

We translated the user requirements into three aspects: instruction, functionality and gameplay. These three elements guided subsequent game design and develpment.



Step 4: Develop Low-fi and High-fi prototyping

The main features of the game was firstly determinated based on the suggestions from experienced corewar players as well as the user cases above. We developed three versions of the prototyps in total. They evolved alongside with the development process.
