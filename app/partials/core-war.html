Problem Description

Core War is a programming PC game firstly introduced in 1984. In the game, two or more battle programs (or we call them "warriors") compete for control of one virtual computer. These warriors are written in an abstract assembly language called Redcode, which is designed for Core War particularly. Therefore fundamental knowledge of assembly language programming is required to play this game.

At the beginning of a game, each battle program is loaded into memory at a random location, after which each program executes one instruction in turn. The objective of the game is to cause the processes of opposing programs to terminate (which happens if they execute an invalid instruction), leaving the victorious program in sole possession of the machine.

Our mission is to bring the game into the mobile era, allowing broader access for knowing, studying and playing this game.

Design Process

1:Understand users

To narrow down the software functionality, we considered various user cases and sketched out personas which represent two typical categories of target users: Core War novice and veteran.

2: Competitive Analysis

As there are no mobile adaptations of the game on either Android or iOS platforms so far, we need to seek our own way for game to develop on the mobile end. The classic version of Core War contains considerable settings as well as functionalities, which give rise to great development complexity and difficulty. Considering the fact that not all the functionalities are necessary for all players, and that we have a time constraint, we decided to simplify the game while keeping its essential logics. To get a better understanding of the game and make a decision on which settings should be remained or removed, we studied the previous software by exmaining and investigating Core War players for advice.
